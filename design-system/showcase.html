Noctua Design System

v2.0 · Ollama-inspired refresh

A re-optimized system for Noctua — the LLM provider compatibility tester. We moved from a dark, high-saturation "tech utility" look to a clean, light-first, near-monochrome aesthetic: calm ink on warm white, the owl's blue reserved as a single accent, and color spent only where the data earns it.

Light-first, ink on warm white

White surfaces, hairline borders, generous air around chrome. Dark theme stays as a refined secondary.

One accent: owl blue

Primary actions are ink black. Blue is for links, focus, and selection — nothing decorative.

Airy chrome, dense data

Nav, cards and headers breathe; tables and JSON stay compact and tabular for real comparison work.

Status earns its color

Green / amber / red / gray appear only on results — accepted, warning, rejected, n/a. Never as decoration.

What changed & next steps — this v2 re-skins the existing token set (same semantic names, so it drops into the current build) toward the Ollama aesthetic: light default, ink primaries, owl-blue accent, gentler radii, softer shadows, Hanken Grotesk + JetBrains Mono. Open items I'd confirm with you: (1) keep both dark + light or ship light-only; (2) whether provider marks stay monochrome ink or carry brand color; (3) whether to apply this across the full app shell (sidebar/topbar) next.
