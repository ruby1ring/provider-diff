测评说明

Noctua 目前聚焦于文本类 LLM 推理协议，在已接入协议下评测流式输出、工具调用与视觉输入三类能力。已支持的文本协议：OpenAI Chat Completions API、Anthropic Messages API；OpenAI Responses API 暂未纳入。

Protocols

支持的协议类型

服务商清单里常见三种 LLM 推理协议。Noctua 目前仅对前两种提供用例库、真实请求跑批与报告导出；OpenAI Responses API 尚未纳入评测范围。

协议 典型 Endpoint 评测状态
OpenAI Chat Completions API POST /v1/chat/completions 已支持
Anthropic Messages API POST /v1/messages 已支持
OpenAI Responses API POST /v1/responses 暂未支持

Overview

什么是协议兼容性测评

配置目标渠道的 API Key、Base URL、Model，从结构化用例库中选择测试集，由后端发起真实 HTTP 请求。每个 case 会输出参数是否被支持、断言是否通过、与 baseline 的结构差异，以及面向网关的处置建议。

Workflow

推荐使用流程

  1. 在测评工具中选择渠道与 endpoint，填写 API Key、Base URL、Model。
  2. 勾选需要验证的参数 case 或组合 case，点击运行测试。
  3. 查看每个 case 的支持结论、断言明细与结构 diff，必要时导出 JSON / Markdown。
  4. 在测评报告中汇总历史跑批，对比供应商兼容性与失败分布。

Output

测评产出

  • 参数/请求是否被供应商支持、拒绝、静默接受但未证明生效
  • 响应是否满足断言（HTTP 状态、必填字段、usage、thinking 证据等）
  • 与 baseline 响应的 JSON 结构差异（字段存在性与类型）
  • 面向网关的处置建议（放行、过滤、适配等）

测评渠道

梳理模型原厂与 API 平台的模型系列覆盖，以及各系列对文本协议的接入情况。

测评模型

主要用于查询新模型（不在测评列表中），例如 GLM5.2。输入模型名称后，会尝试在各测评渠道的模型命名体系中做模糊匹配（大小写、连字符与空格等差异），并列出匹配到的渠道、协议与对应的 API 模型 ID。
